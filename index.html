ESC Coverage

ESCepcion detects every known escalation path

ESC1

Misconfigured Certificate Templates

Templates with Client Auth EKU allow arbitrary SAN + low-priv enrollment — the most exploited ADCS path

ESC2

Any Purpose EKU Abuse

Templates carrying Any Purpose EKU or no EKU at all can authenticate as any domain user

ESC3

Enrollment Agent Abuse

Misconfigured enrollment agent templates allow requesting certificates on behalf of any domain user

ESC4

Template ACL Abuse

Writable ACLs on certificate templates allow low-priv users to introduce ESC1-style misconfigurations

ESC5

PKI Object ACL Abuse

Weak ACLs on CAs, NTAuthCertificates, and Root CA objects can compromise the entire PKI

ESC6

EDITF_ATTRIBUTESUBJECTALTNAME2

CA-level flag allows SAN in any cert request, bypassing all template restrictions entirely

ESC7

Vulnerable CA ACL

ManageCA / ManageCertificates rights enable arbitrary certificate issuance for any domain account

ESC8

NTLM Relay to AD CS HTTP

HTTP enrollment endpoints allow NTLM relay to obtain machine and domain controller certificates

ESC9

No Security Extension

Missing szOID_NTDS_CA_SECURITY_EXT enables certificate-to-account mapping abuse for impersonation

ESC10

Weak Certificate Mappings

Weak CertificateMappingMethods registry settings allow impersonation via strong certificate mappings

ESC11

Unencrypted RPC Interface

IF_ENFORCEENCRYPTICERTREQUEST disabled allows NTLM relay attacks over the RPC certificate interface

ESC12

DCOM Shell Access to CA

Shell access to the CA server via DCOM enables local privilege escalation on the CA host

ESC13

OID Group Link Abuse

Issuance policies linked to universal groups grant certificate holders covert group membership

ESC14

Explicit Certificate Mapping

Explicit cert-to-account mappings writable by low-priv users allow direct account impersonation

ESC15

SchemaVersion 1 / EKUwu

SchemaVersion 1 templates silently ignore EKU restrictions, enabling client auth with any certificate

ESC16

Advanced Detection

Additional escalation paths detected via --deep-scan mode

Quick Start

01

Clone Repository

git clone https://github.com/Hackwarts12/ESCepcion.git
cd ESCepcion
pip install -r requirements.txt
02

Basic Audit

python main.py --domain corp.local --user auditor --dc-ip 192.168.1.10 --password 'P@ssw0rd'
03

Deep Scan

python main.py --domain corp.local --user auditor --dc-ip 192.168.1.10 --password 'P@ssw0rd' --deep-scan

How It Works — The Ritual

1

Discovery

Connects via LDAP and locates all Certificate Authorities within the target domain or forest

→
2

Enumeration

Maps every certificate template, ACL entry, CA configuration flag, and PKI object in the environment

→
3

Analysis

Tests each ESC vector against collected data using rules derived from the latest SpecterOps and Certipy research. The DDCC engine builds attack graphs to separate real threats from theoretical findings

→
4

Reporting

Generates a structured grimoire with findings, exploitability context, attack chains, and remediation instructions
